Git, GitHub e GitLab



O que é

O Git - um Sistema de Controle de Versões. Tipo quando faço alteraçoes em um sistema, etiquetamos aquela alteração, jogando a informação das alterações feitas, e quando essas alterações nao ficaram corretas, podemos retornar a outra versão antes da alteração sem problemas.

O GitHub - é uma plataforma totalmente online e hospedar neles seus projetos, aqui onde o armazenamos as verões de colaborar com softwares open source, seguir outros(as) programadores(as) e interagir com códigos de terceiros. de forma publica ou de forma particular.

O que os dois formam em comum

Bem simples no Git versionamos no GitHub armazenamos, porém de grande utilizade - o GitHub totalmente online - podemos compartilhar os sistemas com diversos coloboradores e todos interagir no mesmo com alteraçoes sem perda de versões Em diversas partes do sistemas por diversos programadores que compartilham um projeto. E evita conflitos de versões tipo se alguem fez a mesma alteração no mesmo lugar existe um controle de alerta.

O GitLab - digamos que seja uma versao mais utilizada para determinados necessidades de forma simples faz a mesma coisa que o GitHub porém para DevOps vai além disso. Tipo, o versionamento criar ramificaçoes. um processo de teste de várias etapas.
